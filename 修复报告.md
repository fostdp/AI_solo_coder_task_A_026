# 问题修复报告

## 🔍 问题定位分析

### 问题1：水位数据异常跳变导致误告警

**现象描述**：
水位计在正常情况下水位从8cm瞬间跳到80cm再立即跳回，触发红色告警，几秒后又自动解除。

**根本原因**：
传感器受到电磁干扰（EMI），导致ADC采样出现瞬时毛刺。后端接收端没有做数据滤波，异常值直接入库并触发告警逻辑。

**问题影响**：
- 频繁误告警，干扰运维人员判断
- 告警记录表产生大量无效告警记录
- 前端地图红点频繁闪烁，用户体验差

---

### 问题2：降雨停止后预测模型仍给出高水位预测

**现象描述**：
降雨突然停止后，系统仍然预测未来30分钟水位会继续上涨，甚至触发预测预警。

**根本原因**：
预测模型只使用了过去1小时累计降雨量，没有考虑降雨状态检测逻辑。即使雨已停，1小时累计值仍然很高，导致模型继续按上涨预测。

**问题影响**：
- 预测预警误报
- 实际水位下降时仍显示预测预警
- 降低预测可信度

---

## ✅ 修复方案

### 修复1：水位数据跳变滤波

**改动点**：`server.js:44-70`

新增函数 `validateWaterLevel()`

```javascript
async function validateWaterLevel(pointId, newLevel, maxJump = 20)
```

**过滤逻辑**：
1. 查询该点位上一条有效水位记录
2. 计算新旧水位差值绝对值
3. 差值 > 20cm 时丢弃该数据点
4. 返回上一条有效值继续后续计算

**改动范围**：
- `server.js:44-70` 新增滤波函数
- `server.js:200-210` 在数据接收API中调用
- API响应新增过滤结果字段

**数据库改动**：无

---

### 修复2：降雨停止检测与预测衰减

**改动点**：`server.js:72-107`

**新增函数**：

1. `checkRainStopped(pointId)` - 检测降雨是否停止
```javascript
查询最近10分钟降雨量 < 0.5mm 判定为雨停
```

2. `predictWaterLevel()` 增强版
```javascript
function predictWaterLevel(currentLevel, rainfallLastHour, changeRate, isRainStopped)
```

**雨停时衰减逻辑**：
- 降雨影响系数 × 0.3（衰减系数）
- 变化速率影响强制归零或反向
- 正增长速率时强制转为下降预测

**改动范围**：
- `server.js:72-82` 新增降雨停止检测函数
- `server.js:84-107` 预测函数增强（新增 isRainStopped 参数）
- `server.js:239` 调用降雨检测
- `server.js:241` 预测时传入雨停状态
- `init.sql:68` predictions表新增rain_stopped字段
